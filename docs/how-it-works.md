# Как работает NetBypass

## Принцип

Файл `hosts` позволяет Windows сопоставить доменное имя с IP-адресом до
обращения к обычному DNS-серверу. NetBypass собирает записи выбранных сервисных
модулей и добавляет их в собственный маркированный блок.

Это не VPN:

- трафик не шифруется дополнительным туннелем;
- внешний IP пользователя не меняется;
- программа не перенаправляет весь сетевой трафик;
- после применения NetBypass не должен оставаться запущенным.

## Состояния

При каждом запуске приложение читает реальный системный `hosts`:

- `Inactive` — маркеров NetBypass нет;
- `Active` — блок существует и совпадает с выбранными модулями;
- `ChangesPending` — блок существует, но его содержимое отличается от выбора;
- `Corrupted` — найден только один маркер, неверный порядок или дубликаты.

Сохранённый логический флаг не используется, поэтому состояние корректно
определяется после закрытия приложения и перезагрузки Windows.

## Запись файла

Перед первым изменением создаётся резервная копия. Затем NetBypass:

1. читает существующий файл;
2. удаляет только прежний управляемый блок;
3. добавляет новый блок;
4. пытается атомарно заменить файл;
5. при запрете замены записывает содержимое в существующий файл;
6. перечитывает файл и проверяет результат;
7. очищает DNS-кеш Windows.

Пользовательские строки вне управляемого блока сохраняются.

## Хранение настроек

Выбранные идентификаторы модулей сохраняются в:

```text
%LOCALAPPDATA%\NetBypass\settings.json
```

Системный эффект определяется не этим файлом, а фактическим содержимым
`hosts`.
