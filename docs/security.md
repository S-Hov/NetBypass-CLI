# Безопасность и ограничения

## Доверие к IP-адресам

Запись `hosts` направляет домен на явно указанный сервер. Вредоносный или
скомпрометированный модуль способен перенаправить пользователя на чужой IP.

Перед добавлением или обновлением модуля необходимо проверять:

- источник списка;
- владельца IP-адреса;
- назначение каждого домена;
- изменения относительно предыдущей версии.

HTTPS продолжает проверять сертификат сайта, но это не заменяет аудит модулей.

## Права администратора

Приложение запрашивает повышение при запуске. Это требуется для изменения
системного файла. Остальная логика не устанавливает драйверы, службы или
сетевые адаптеры.

## Что NetBypass не делает

- не скрывает IP пользователя;
- не шифрует трафик как VPN;
- не защищает в публичных Wi-Fi;
- не обходит блокировки по IP, SNI или DPI;
- не гарантирует доступность всех функций сервиса;
- не обновляет IP автоматически из непроверенных источников.

## Сохранение чужих записей

Обычное включение и отключение работают только с маркированным блоком.
Восстановление повреждённого блока удаляет известные заголовки и записи
модулей. Полная резервная копия сохраняется отдельно, но автоматически поверх
текущего файла не восстанавливается.

## Ответственное использование

Пользователь самостоятельно отвечает за соблюдение законодательства,
корпоративных правил и условий использования сетевых сервисов.
