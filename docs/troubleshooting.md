# Устранение проблем

## Access to path is denied

Убедитесь, что в запросе UAC было выбрано **Да**. NetBypass должен работать с
правами администратора.

Если ошибка сохраняется:

1. закройте другие редакторы файла `hosts`;
2. временно проверьте журнал или защиту антивируса;
3. убедитесь, что файл не имеет атрибута «Только чтение»;
4. запустите приложение повторно.

NetBypass сначала пытается заменить файл атомарно, а при запрете заменяет
содержимое существующего файла.

## Статус «Файл hosts требует внимания»

Такой статус появляется, если:

- присутствует только один маркер NetBypass;
- конечный маркер расположен раньше начального;
- маркеры встречаются несколько раз.

Используйте **Восстановить hosts**. Приложение удалит известные строки
NetBypass и сохранит остальные записи.

## Сервис не открывается

Возможные причины:

- IP в модуле устарел;
- сервис использует дополнительные домены;
- провайдер применяет блокировку не на уровне DNS;
- браузер использует собственный DNS-кеш;
- антивирус или сетевой фильтр перехватывает соединение.

Попробуйте перезапустить браузер. Если это не помогает, проверьте актуальность
модуля. NetBypass не способен обходить все виды блокировок.

## Настройки изменены, но эффекта нет

После изменения галочек нажмите **Сохранить и применить**. Простое изменение
выбора не перезаписывает активный `hosts`.

## Ручное восстановление

Исходная резервная копия находится в:

```text
%LOCALAPPDATA%\NetBypass\Backups\hosts.initial.bak
```

Полная ручная замена системного файла этой копией удалит изменения, сделанные
другими программами после её создания. Используйте её только осознанно.
