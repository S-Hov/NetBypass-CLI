# Сервисные модули

Каждый сервис хранится в отдельном файле:

```text
src/NetBypass.App/Modules/<id>.hosts
```

Это позволяет независимо включать сервисы и обновлять их домены.

## Формат

```text
# id: openai
# name: OpenAI / ChatGPT
# category: AI
# default: false

45.155.204.190 chatgpt.com
45.155.204.190 api.openai.com
```

Обязательные поля:

- `id` — уникальный стабильный идентификатор;
- `name` — название в интерфейсе;
- `category` — группа на странице сервисов.

Поле `default` поддерживается форматом, но текущий интерфейс также использует
собственный список модулей, отключённых при первом запуске.

## Проверка безопасности

Загрузчик отклоняет:

- `0.0.0.0`;
- loopback-адреса;
- private- и link-local-адреса;
- multicast-адреса;
- `localhost` и поддомены `.localhost`;
- строки без корректной пары `IP домен`.

Большие рекламные и вредоносные блок-листы не относятся к задаче NetBypass и
не должны добавляться в сервисные модули.

## Добавление сервиса

1. Создайте отдельный `.hosts`-файл.
2. Укажите уникальный `id`, понятное имя и категорию.
3. Добавьте только необходимые домены.
4. Проверьте источник и владельца IP-адресов.
5. Выполните `dotnet test`.

Не объединяйте несвязанные сервисы в один модуль без технической необходимости.
