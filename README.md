# NetBypass

NetBypass — Windows-утилита для удобного управления доступом к выбранным
сервисам. Сейчас приложение работает через проверенные записи `hosts`, а
дальше развивается в управляющий центр для DNS, релеев и внешних Anti-DPI
движков вроде GoodbyeDPI, ByeDPI и zapret.

> NetBypass не является VPN и не создаёт сетевой туннель. Он помогает только
> выбранным сервисам и должен показывать, какая стратегия применена.

## Возможности

- включение и отключение одной большой кнопкой;
- отдельный выбор сервисов: OpenAI, Claude, Google, Twitch и другие;
- определение реального состояния `hosts` при каждом запуске;
- проверка DoH, TCP, TLS и HTTP перед применением адресов;
- отдельная страница диагностики для выбранных сервисов;
- сохранение пользовательских и системных записей;
- безопасное удаление только блока NetBypass;
- резервная копия исходного `hosts`;
- защита от локальных, loopback- и некорректных адресов;
- автоматическая очистка DNS-кеша Windows.

## Целевое направление

NetBypass не планируется как собственный Anti-DPI-движок с нуля. Цель проекта —
сделать понятную оболочку над несколькими стратегиями:

- проблема с DNS → DoH или локальный DNS;
- нужен другой регион → релей;
- блокировка через DPI → внешний движок GoodbyeDPI, ByeDPI или zapret.

Пользователь выбирает сервисы, а NetBypass диагностирует проблему, выбирает
подходящий способ, запускает нужный адаптер и проверяет, что после отключения
ничего не осталось.

## Как скачать и запустить

Готовые версии будут размещаться в разделе **Releases** репозитория. Если
готовой сборки ещё нет, приложение можно запустить из исходного кода.

### Запуск из исходного кода

Потребуются:

- Windows 10 или Windows 11;
- [.NET 10 SDK](https://dotnet.microsoft.com/download/dotnet/10.0);
- Git — необязательно, архив можно скачать через **Code → Download ZIP**.

Клонируйте репозиторий или распакуйте ZIP, затем откройте PowerShell в папке
проекта:

```powershell
dotnet restore
dotnet build
dotnet run --project .\src\NetBypass.App
```

Windows покажет запрос контроля учётных записей. Права администратора нужны
только для работы с системным файлом `hosts`.

## Как пользоваться

1. Откройте страницу **Сервисы** и проверьте выбранные сайты.
2. Нажмите **Сохранить и применить** либо вернитесь на главную и нажмите
   большую кнопку **Включить**.
3. После статуса **Защита активна** приложение можно закрыть.
4. Чтобы отменить изменения, снова нажмите большую кнопку.
5. При проблемах используйте **Восстановить hosts** — будут удалены только
   записи NetBypass.

NetBypass при следующем запуске сам проверит файл и покажет, активны ли записи,
изменились ли настройки или повреждён управляемый блок.

## Важно знать

- IP-адреса сервисов могут меняться, поэтому модули требуют обновления.
- Метод не помогает при блокировках по IP, SNI или DPI.
- Антивирус может контролировать изменение `hosts`.
- Не скачивайте модули из непроверенных источников: они определяют, на какие
  IP-адреса будут направлены домены.

## Статус проекта

Проект находится в ранней разработке. Текущая версия реализует базовый
`hosts`-режим и диагностику. Интеграция внешних Anti-DPI-движков ещё не
реализована и будет добавляться только после появления безопасного журнала
ресурсов и полной очистки после отключения.
